🔐 VM SECURITY RISKS P1 · MOST TESTED
🔓
VM Escaping
  • Malware breaks out of guest OS
  • Accesses hypervisor or host OS
  • Can infect other VMs on same host
  • Most severe virtualization threat
📂
VM Sprawl
  • Too many unmanaged VMs created
  • No documentation or tracking
  • VMs go unpatched — security risk
  • Resources wasted, inventory lost
📸
Snapshot Risks
  • Snapshots capture VM state
  • Old snapshots = outdated patches
  • Rolling back = reintroducing vulns
  • Must manage snapshot lifecycle
🧠 MEMORY CUE
VM Escaping = The prisoner breaks out of their cell and reaches the warden's office (host)
VM Sprawl = Hotel that keeps adding rooms with no staff or cleaning schedule
❓ Which of the following is the PRIMARY security risk where malware in one guest VM can access the host OS or other VMs?
❓ An IT team keeps creating VMs without documentation or patching schedules. What problem is this?
⚡ EXAM TRAPS
⚡ VM Escaping ≠ just a software bug — it's a full isolation breach
⚡ Snapshots are NOT automatically safe — rolling back reintroduces old vulnerabilities
⚡ VM Sprawl is an ADMINISTRATIVE problem, not a technical failure
🖥️ HOST REQUIREMENTS P3 · BIOS CRITICAL
💾
Memory (RAM)
  • Every VM consumes dedicated RAM
  • Most common bottleneck on host
  • Host + all guests share physical RAM
  • Insufficient RAM = poor VM performance
⚙️
CPU Virtualization
  • Intel: VT-x (Virtualization Technology)
  • AMD: AMD-V (Virtualization)
  • Must be enabled in BIOS/UEFI
  • Without it: hypervisors cannot start
💿
Storage
  • Each VM has its own virtual disk
  • SSD preferred for VM performance
  • Storage pools span multiple drives
  • RAID adds redundancy for VMs
VT-x / AMD-V REQUIREMENT FLOW
❓ What is the MOST common hardware bottleneck when running multiple virtual machines on a single host?
❓ Which BIOS/UEFI setting MUST be enabled for a hypervisor to run virtual machines?
⚡ EXAM TRAPS
⚡ CPU may SUPPORT virtualization but BIOS has it disabled by default — always check BIOS
⚡ Memory is the bottleneck, NOT CPU — every guest VM consumes dedicated RAM
⚡ VT-x = Intel · AMD-V = AMD — both do the same thing, different name per brand

☁️ CLOUD SERVICE MODELS P5 · CLASSIFY FAST
IaaS
Infrastructure as a Service
Virtual servers, storage, networking — YOU manage OS + apps
PaaS
Platform as a Service
Development environment — build/deploy apps without managing OS
SaaS
Software as a Service
Subscription software delivered via internet — email, CRM, Office365
DaaS
Desktop as a Service
Virtual desktops streamed to end users — hosted Windows/Linux desktops
🧠 MEMORY STACK — Who Controls What?
IaaS → You control: OS, apps, data (most control)
PaaS → You control: apps, data only
SaaS → You control: data only (least control)
DaaS → You control: user desktop experience
🔵 IaaS vs PaaS
•IaaS: raw infra — VMs, storage, networks
•PaaS: dev platform — runtime, tools, middleware
•IaaS = DevOps / Sysadmin use
•PaaS = Developers building apps
VMsStorageDev Tools
🟡 SaaS vs DaaS
•SaaS: software you use in a browser
•DaaS: full desktop OS streamed to you
•SaaS = Google Docs, Office 365, Salesforce
•DaaS = Virtual Desktop Infrastructure (VDI)
SubscriptionBrowserVDI
❓ A company provisions virtual servers, load balancers, and cloud storage. Which cloud service model are they using?
❓ A development team needs a cloud environment to build and deploy apps without managing the underlying OS. Which model?
⚡ EXAM TRAPS
⚡ "Developer environment" = PaaS — NOT IaaS even if it's in the cloud
⚡ "Subscription software" = SaaS — don't confuse with DaaS
⚡ DaaS ≠ SaaS — DaaS is a full DESKTOP, SaaS is just an APP
⚡ IaaS = most flexibility/control · SaaS = least control

📈 CLOUD CHARACTERISTICS P6 · ELASTICITY VS SCALABILITY
🔵 Rapid Elasticity
•AUTOMATIC resource adjustment
•Scales UP during traffic spike
•Scales DOWN when demand drops
•Responds in real-time
•Think: rubber band stretching
DynamicAutoReal-time
🟡 Scalability
•Ability to GROW capacity
•Handles more users over time
•Planned capacity expansion
•Does NOT auto-shrink
•Think: adding more lanes to a highway
GrowthPlannedUsers
🔗
On-Demand Self-Service
  • Provision resources without human interaction
  • Automated via portal or API
  • No waiting for IT approval
📡
Broad Network Access
  • Access from any device, anywhere
  • Laptops, phones, tablets
  • Via internet or private WAN
🔄
Resource Pooling
  • CSP shares physical resources
  • Multi-tenant model
  • Customers unaware of exact location
📊
Measured Service
  • Pay-per-use metering
  • Monitor, control, report usage
  • Transparent billing model
❓ A cloud platform automatically increases server resources during a flash sale and reduces them afterward. Which cloud characteristic is this?

⚡ Elasticity = AUTOMATIC up AND down — Scalability = planned growth capacity
⚡ "Traffic spikes" = Elasticity · "More users over time" = Scalability
⚡ Scalability does NOT automatically reduce resources when demand drops
