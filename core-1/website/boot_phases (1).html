🚨 Student Errors Most common exam mistakes and why they happen ▼
1
"Bootmgr missing" → student runs bootrec /fixmbr
Wrong command. fixmbr fixes the MBR map — but the map already worked. The problem is the file it pointed to is gone. You need /rebuildbcd.
🔧 Fix: bootrec /rebuildbcd
2
"No Bootable Device" → student checks cables first
Cables are only the answer if the drive doesn't appear in UEFI at all. If the drive IS listed but won't boot — MBR is corrupt. Cables are fine.
🔧 First: does the drive appear in UEFI list? Yes → bootrec. No → check cable.
3
Linux fails "Security Violation" → student reinstalls
Reinstalling does nothing. Secure Boot is rejecting the unsigned bootloader. One UEFI setting fixes it.
🔧 Fix: UEFI → Secure Boot → Disabled
4
BSOD 0x0000007B → student replaces the drive
The drive is fine. SATA mode was changed AHCI↔IDE in UEFI. Replacing the drive changes nothing.
🔧 Fix: UEFI → SATA Mode → AHCI
5
"No Bootable Device" with Secure Boot/Legacy mismatch → student runs bootrec
The drive and MBR are fine. UEFI sees the drive but refuses it because of a mode conflict. bootrec fixes nothing here.
🔧 Fix: Match UEFI mode to drive format — enable UEFI mode or convert to GPT
6
Date resets every boot → student clears CMOS
Clearing CMOS wipes settings but doesn't replace the dead battery. Date resets again next boot.
🔧 Fix: Physically replace the CR2032 coin battery
